# GitHub 搜索与迁移承接

本文件记录 LibreTV Revival 的 GitHub 搜索和迁移承接策略。目标是让来自已归档上游 `LibreSpark/LibreTV` 的自部署用户能通过仓库、release、Discussions 和 Issues 找到维护版，而不是导向任何公开播放站点。

## 搜索定位

仓库描述、topics、README 和 release notes 应稳定覆盖这些关键词：

- LibreTV Revival
- LibreSpark/LibreTV archived fork
- self-hosted LibreTV
- Cloudflare Pages
- Cloudflare Pages Functions proxy
- PWA
- MacCMS VOD API
- source health checks
- playback URL fixes
- diagnostics

## 承接入口

- Discussions：承接迁移、配置、部署平台、源选择和排障前置讨论。
- Issues：只承接可复现 bug、明确功能请求和文档缺口。
- Releases：面向 GitHub 搜索和 watch 用户，记录稳定能力和迁移入口。
- README：首屏说明维护续作定位、上游归档背景、迁移入口和安全边界。

## 安全边界

- 不写公开部署地址。
- 不宣传成公共影视服务。
- 不把成人源作为卖点。
- 不要求用户公开密码、令牌、私人源、完整代理链接或实例地址。
